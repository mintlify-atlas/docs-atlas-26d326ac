---
title: Ingest
description: Start ingestion sessions from web URLs, manual text, or file uploads.
---

Ingestion creates a **draft session** in Redis. The session contains auto-chunked content that you review and edit in the RAGler UI before publishing to a collection.

**Base path:** `/api/ingest`

**Required header:** `X-User-ID`

---

## Ingest from a web URL

```http
POST /api/ingest/web
```

<Note>
This endpoint requires the `webIngest` feature flag to be enabled. If disabled, the endpoint returns `403 Forbidden`.
</Note>

Fetches content from the given URL using Mozilla Readability, sanitises it, then runs the chunking pipeline. Private IPs and localhost are blocked (SSRF protection).

<ParamField body="url" type="string" required>
  Public URL to ingest. Must be a valid HTTP/HTTPS URL.
</ParamField>

<ParamField body="chunkingConfig" type="object">
  Optional chunking configuration. If omitted, defaults apply.

  <Expandable title="chunkingConfig fields">
    <ParamField body="chunkingConfig.method" type="string">
      Chunking strategy. `llm` (default) uses the LLM to determine natural chunk boundaries.
      `character` splits on character count.
    </ParamField>
    <ParamField body="chunkingConfig.chunkSize" type="number">
      Target chunk size in characters. Min 100, max 10,000. Default `1000`.
    </ParamField>
    <ParamField body="chunkingConfig.overlap" type="number">
      Character overlap between adjacent chunks. Min 0, max 2,000. Default `200`.
    </ParamField>
  </Expandable>
</ParamField>

```bash
curl -X POST http://localhost:3000/api/ingest/web \
  -H "Content-Type: application/json" \
  -H "X-User-ID: user-123" \
  -d '{
    "url": "https://docs.example.com/getting-started",
    "chunkingConfig": {
      "method": "llm",
      "chunkSize": 800,
      "overlap": 100
    }
  }'
```

**Response `201`**

```json
{
  "sessionId": "sess-a1b2c3d4",
  "sourceType": "web",
  "sourceUrl": "https://docs.example.com/getting-started",
  "status": "DRAFT",
  "createdAt": "2024-01-15T10:00:00.000Z"
}
```

<Expandable title="Response fields">
  <ResponseField name="sessionId" type="string">
    Unique session identifier. Use this with the Session API to edit and publish chunks.
  </ResponseField>
  <ResponseField name="sourceType" type="string">
    Always `web` for this endpoint.
  </ResponseField>
  <ResponseField name="sourceUrl" type="string">
    The URL that was ingested.
  </ResponseField>
  <ResponseField name="status" type="string">
    Initial session status. Always `DRAFT` on creation.
  </ResponseField>
  <ResponseField name="createdAt" type="string">
    ISO-8601 timestamp of session creation.
  </ResponseField>
</Expandable>

## Ingest manual text

```http
POST /api/ingest/manual
```

Ingest raw text content directly. Useful for pasting notes, documentation snippets, or any text that does not have a public URL.

<ParamField body="content" type="string" required>
  The text content to ingest. Min 1 character, max 100 KB (102,400 characters).
</ParamField>

<ParamField body="chunkingConfig" type="object">
  Optional chunking configuration (same fields as web ingest above).
</ParamField>

```bash
curl -X POST http://localhost:3000/api/ingest/manual \
  -H "Content-Type: application/json" \
  -H "X-User-ID: user-123" \
  -d '{
    "content": "RAGler is a knowledge operations platform that puts humans in the loop of RAG pipelines...",
    "chunkingConfig": {
      "method": "llm"
    }
  }'
```

**Response `201`** — same shape as web ingest, with `sourceType: "manual"`.

## Ingest a file

```http
POST /api/ingest/file
```

<Note>
This endpoint requires the `fileIngest` feature flag to be enabled.
</Note>

Upload a file for ingestion. The file is parsed server-side and then chunked.

**Content-Type:** `multipart/form-data`

**Supported file types:** `.pdf`, `.txt`, `.md`, `.csv`, `.json`, `.html`, `.htm`, `.xml`, `.docx`, `.xlsx`

**Maximum file size:** 10 MB

<ParamField body="file" type="file" required>
  The file to upload. Must use the field name `file`.
</ParamField>

```bash
curl -X POST http://localhost:3000/api/ingest/file \
  -H "X-User-ID: user-123" \
  -F "file=@/path/to/document.pdf"
```

**Response `201`** — same shape as web ingest, with `sourceType: "file"` and `sourceUrl` set to an internal file reference.

**Response `400`** — if no file is provided or the file extension is not supported.

<Warning>
Unsupported file types return a `400 Bad Request` with a message listing the supported extensions.
</Warning>
